Overview

Entry point for the CXP library. This class provides convenient methods to save some boiler-plate code, and also to provide access to otherwise private or protected APIs

+ initialize:error:

Initializes the CXP internal states and prepare the proper functioning of subsequent methods.

+ (BOOL)initialize:(NSString *)configurationPath error:(NSError **)error

Parameters

configurationPath

The file path containing the configuration information.
error

If an error occurs, upon return contains an NSError object that describes the problem.

Return Value

Yes if the objects could be initialized properly. No otherwise.

Discussion

Initializes the CXP internal states and prepare the proper functioning of subsequent methods.

By default the configuration file could be encrypted or not. This method will attempt to decrypt the file and it will fall back to read as plain if the decryption failed. To override this behavior use the initialize:forceDecryption:error: instead

+ startPerformanceEvent:withObjectId:

Starts a performance event to be queued and wait for the end signal to measure the time spend in this operation.

+ (void)startPerformanceEvent:(NSString *)operation withObjectId:(NSString *)objId

Parameters

operation

The name of the operation that is starting. This exact same name has to be send to end this operation.
objId

Optional object that is requesting an action. If no object is associated with this action nil can be sent.

Discussion

Starts a performance event to be queued and wait for the end signal to measure the time spend in this operation.

It’s possible to start this event performances from javascript widgets. In order to do so, send a pubsub gadgets.pubsub.publish(“cxp.performance.start”, {“operation”:“operation”, “id”: “objId”}) As well as in this implementation, operation is mandatory and the id is optional.

Targeting Methods

+ addTargetingParameter:forKey:

Adds a targeting parameter to be send to the server when the model is requested. The parameter will be added as a header with the prefix X-Targeting-, to allow easier filtering/parsing on the server collector. For instance, calling this method with key=“myparam” value=“3.14” will result in a header called

X-Targeting-myparam: 3.14
+ (NSString *)addTargetingParameter:(NSString *)value forKey:(NSString *)key

Parameters

value

The value to be send as the parameter
key

The name of the parameter

Return Value

The previous value on the same key, otherwise nil

Discussion

Adds a targeting parameter to be send to the server when the model is requested. The parameter will be added as a header with the prefix X-Targeting-, to allow easier filtering/parsing on the server collector. For instance, calling this method with key=“myparam” value=“3.14” will result in a header called

X-Targeting-myparam: 3.14

Not all characters are valid as a key. For this reason, colons (:) are replaced with dashes (-), and space (0x20) are replaced with underscore (_).
