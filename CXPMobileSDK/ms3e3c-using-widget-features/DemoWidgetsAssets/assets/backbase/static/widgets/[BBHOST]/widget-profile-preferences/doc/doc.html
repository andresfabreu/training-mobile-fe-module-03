Profile Preferences

Introduction



The Preferences widget provides a very unique set of functionality, which allows the user to make application wide changes to how the site is presented to the user. The following features are customizable by the end user:

  • Preferred Name - How the application will address the user
  • Preferred Language - The default language to display the portal in, as chosen by the user
  • Default View - When a user logs in, go to the home page, or directly to Springboard
  • Default Account - The account to be automatically selected when a user opens up a widget
  • Default Balance - unless otherwise specified, this is the balance the user will see whenever that information is displayed anywhere

Configuration

Provide a table detailing the list of configurable properties available to the widget:

Preference Description
accountsDataSrc The URL endpoint to retrieve the list of possible accounts to select as the default account
preferredNameService The URL endpoint that receives a POST request in order to update the users Preferred Name
